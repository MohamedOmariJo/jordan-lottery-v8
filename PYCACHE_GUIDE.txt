❌ لا، لا يجب رفع مجلدات __pycache__ إلى GitHub!

═══════════════════════════════════════════════════════════════
🤔 ما هي مجلدات __pycache__؟
═══════════════════════════════════════════════════════════════

__pycache__ هي مجلدات يُنشئها Python تلقائياً لتخزين:
- ملفات .pyc (Python bytecode)
- نسخ مُترجمة مسبقاً من ملفات .py
- تسريع تحميل الملفات في المرات القادمة

مثال:
config/
├── __init__.py
├── settings.py
└── __pycache__/           ← يُنشأ تلقائياً
    ├── __init__.cpython-313.pyc
    └── settings.cpython-313.pyc

═══════════════════════════════════════════════════════════════
❌ لماذا لا يجب رفعها؟
═══════════════════════════════════════════════════════════════

1. ✗ ليست جزءاً من الكود المصدري
2. ✗ تُنشأ تلقائياً عند تشغيل Python
3. ✗ خاصة بنظام التشغيل ونسخة Python
4. ✗ تزيد حجم المستودع بدون فائدة
5. ✗ قد تسبب مشاكل عند النشر

═══════════════════════════════════════════════════════════════
✅ كيف أتجنب رفعها؟
═══════════════════════════════════════════════════════════════

الطريقة 1️⃣: حذفها قبل الرفع
────────────────────────────────

في Windows (PowerShell):
Get-ChildItem -Path . -Recurse -Filter __pycache__ | Remove-Item -Recurse -Force

في Linux/Mac:
find . -type d -name __pycache__ -exec rm -rf {} +

─────────────────────────────────────────

🔧 أوامر سريعة للحذف:
═══════════════════════════════════════════════════════════════

Windows (Command Prompt):
for /d /r . %d in (__pycache__) do @if exist "%d" rd /s /q "%d"

Windows (PowerShell):
Get-ChildItem -Recurse -Directory -Filter __pycache__ | Remove-Item -Recurse -Force

Linux/Mac:
find . -type d -name __pycache__ -exec rm -rf {} + 2>/dev/null

Git (إذا رفعتها بالخطأ):
